# PlayTime Tracker

Серверный мод для **NeoForge 1.21.1**, который отслеживает игровое время игроков и сохраняет статистику в **PostgreSQL**.

Мод работает в фоне: фиксирует вход и выход игроков, записывает завершённые сессии и обновляет агрегированную статистику. Игровых команд и клиентской части нет — только сервер.

## Возможности

- Учёт сессий: время входа, выхода и длительность в секундах
- **Общее игровое время** (`total_playtime_seconds`) — накапливается при каждом выходе
- **Время за последние 7 дней** (`weekly_playtime_seconds`) — пересчитывается из истории сессий с учётом пересечения интервалов (скользящее окно 7 суток)
- **Heartbeat сессий** — каждые 5 минут обновляет метку последней активности в БД
- **Восстановление после краша** — при следующем запуске сервера незакрытые сессии автоматически завершаются по времени последнего heartbeat
- Автоматическое создание и миграция таблиц в БД при старте сервера
- Асинхронная запись в PostgreSQL (не блокирует главный поток сервера)
- Сохранение активных сессий при корректной остановке сервера (`/stop`)

## Требования

| Компонент | Версия |
|-----------|--------|
| Minecraft | 1.21.1 |
| NeoForge | 21.1+ |
| PostgreSQL | любая актуальная версия с поддержкой `UUID` и `TIMESTAMPTZ` |
| Сторона | только **сервер** (на клиент ставить не нужно) |

## Установка

1. Установите NeoForge 1.21.1 на сервер.
2. Соберите мод или возьмите готовый JAR из `build/libs/`.
3. Положите JAR в папку `mods/` сервера.
4. Подготовьте базу PostgreSQL (достаточно пустой БД — таблицы создаст сам мод).
5. Создайте и заполните `database.properties` (см. раздел [Настройка](#настройка)) **до** первого запуска — так не придётся перезапускать сервер из‑за дефолтного конфига.
6. Запустите сервер.

## Настройка

### Куда положить `database.properties`

Файл читается **от корня сервера** — из той папки, откуда запускается Minecraft (там же, где лежат `mods/`, `world/`, `server.properties`).

```
<корень сервера>/
├── mods/
│   └── playtime_tracker-....jar
├── config/
│   └── playtime_tracker/
│       └── database.properties    ← сюда
├── server.properties
└── ...
```

Полный путь к файлу:

```text
config/playtime_tracker/database.properties
```

На Windows, например:

```text
D:\MinecraftServer\config\playtime_tracker\database.properties
```

Папку `config/playtime_tracker/` можно создать вручную до первого запуска. Если её нет, мод создаст её сам при старте.

### Как заполнить файл

Создайте `database.properties` в указанной папке со следующим содержимым:

```properties
host=localhost
port=5432
database=playtime_tracker
username=postgres
password=change_me
```

Подставьте свои значения для PostgreSQL. **Обязательно смените пароль** перед production-использованием.

| Параметр | Описание |
|----------|----------|
| `host` | Адрес сервера PostgreSQL |
| `port` | Порт (по умолчанию 5432) |
| `database` | Имя базы данных |
| `username` | Пользователь БД |
| `password` | Пароль |

### Два способа настройки

**Рекомендуется:** создать `config/playtime_tracker/database.properties` вручную **до** первого запуска сервера с модом. Тогда мод сразу подключится к вашей БД.

**Альтернатива:** запустить сервер без файла — мод создаст шаблон с дефолтными значениями, но подключение к БД, скорее всего, не удастся. Остановите сервер, отредактируйте созданный файл и запустите снова.

При локальной разработке через `./gradlew runServer` корень сервера — папка `run/` в каталоге проекта (путь к конфигу: `run/config/playtime_tracker/database.properties`).

## Как это работает

```
Игрок зашёл   →  сессия создаётся в БД и запоминается в памяти
Каждые 5 мин  →  heartbeat обновляет last_seen_at в БД
Игрок вышел   →  сессия закрывается, обновляется total и weekly
/stop         →  все активные сессии сохраняются, пул соединений закрывается
Краш / рестарт →  при старте незакрытые сессии завершаются по last_seen_at
```

### Heartbeat и восстановление

Пока игрок на сервере, мод раз в **5 минут** записывает в БД время последней активности (`last_seen_at`). Если сервер упал без `/stop`, при следующем запуске мод находит все открытые сессии (`logout_time IS NULL`) и закрывает их, считая момент выхода равным `last_seen_at`.

Так теряется не более **~5 минут** игрового времени с момента последнего heartbeat. Если краш произошёл раньше первого heartbeat после входа, сессия закроется по времени входа (`login_time`).

При повторном входе игрока мод сначала закрывает возможную «осиротевшую» сессию этого игрока, затем открывает новую.

### Таблицы в PostgreSQL

**`player_sessions`** — история сессий (источник правды для weekly):

| Столбец | Тип | Описание |
|---------|-----|----------|
| `id` | BIGSERIAL | Первичный ключ |
| `player_uuid` | UUID | UUID игрока |
| `login_time` | TIMESTAMPTZ | Время входа |
| `last_seen_at` | TIMESTAMPTZ | Время последнего heartbeat (или входа, если heartbeat ещё не был) |
| `logout_time` | TIMESTAMPTZ | Время выхода (`NULL` — сессия ещё открыта) |
| `duration_seconds` | BIGINT | Длительность сессии в секундах |

На уровне БД гарантируется не более одной открытой сессии на игрока (частичный уникальный индекс по `player_uuid` при `logout_time IS NULL`).

**`player_playtime`** — агрегированная статистика:

| Столбец | Тип | Описание |
|---------|-----|----------|
| `player_uuid` | UUID | Первичный ключ |
| `total_playtime_seconds` | BIGINT | Общее время за всё время |
| `weekly_playtime_seconds` | BIGINT | Время за последние 7 дней |

Запись в `player_playtime` создаётся при **первом выходе** игрока. Пока игрок ни разу не выходил, строки для него в таблице нет.

### Подсчёт weekly

Время за 7 дней считается не по `login_time`, а по **пересечению** сессии с окном `[сейчас − 7 дней, сейчас]`. Например, сессия с 23:00 до 03:00 даст в weekly только часы после полуночи, а не всю сессию целиком.

## Использование

Мод не требует действий от игроков и администраторов в игре. После настройки БД он работает автоматически.

Данные можно смотреть напрямую в PostgreSQL:

```sql
-- Статистика игрока
SELECT * FROM player_playtime WHERE player_uuid = '...';

-- История сессий
SELECT * FROM player_sessions WHERE player_uuid = '...' ORDER BY login_time DESC;

-- Текущие открытые сессии
SELECT * FROM player_sessions WHERE logout_time IS NULL;
```

UUID игрока можно узнать, например, через логи сервера или внешние инструменты.

## Ограничения

- При **краше** сервера (kill -9, OOM, обрыв питания) время сессии сохраняется только до последнего heartbeat — возможна потеря до **~5 минут** с момента последней записи.
- Задачи записи в БД, ещё не выполненные на момент краша (например, только что открытая сессия), **могут не сохраниться**.
- Корректная остановка через `/stop` сохраняет все активные сессии полностью.
